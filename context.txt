1. PROJECT OVERVIEW
================================================================================

LAMDA is a production-ready Supply Chain Risk Prediction System that uses
AI-powered intelligence processing, Graph Neural Networks (GNN) for spatial
risk propagation, and multi-objective route optimization to assess and mitigate
supply chain risks in real time.

The system processes data from 6 different scraper sources, uses Claude AI
(Anthropic) for semantic analysis of unstructured text, propagates risks
through a Graph Attention Network (GAT), and finds optimal shipping routes
using A* pathfinding with a multi-objective cost function.

Target Users:
  - Logistics companies needing real-time route risk assessment
  - Supply chain managers monitoring global disruption risks
  - Operations teams optimizing shipping routes across ports/cities

Key Capabilities:
  - AI-Powered Intelligence Processing (Claude API for news/political/weather)
  - Graph Neural Network risk propagation (GAT with PyTorch Geometric)
  - Temporal Memory (tracks risk history for trend analysis, last 10 updates)
  - Multi-Objective Route Optimization (balances risk, distance, time, trade)
  - REST API (Flask-based, easy integration with web/mobile frontends)
  - Automated Updates (background worker updates graph every 30 minutes)
  - Database Persistence (SQLite with risk history)

    - Formula: adjusted_risk = base_risk * credibility + 0.5 * (1 - credibility)

4.3  route_optimizer.py  (341 lines)
     Purpose: A* pathfinding with multi-objective cost optimization
--------------------------------------------------------------------------------

IMPORTS:
  json, heapq, numpy, typing, dataclasses, math (radians, cos, sin, asin, sqrt)

DATA CLASSES:

  Route (dataclass):
    - path: List[str]             — Ordered list of node IDs
    - total_distance_km: float
    - total_risk: float           — Sum of node risks along path
    - avg_risk: float             — total_risk / number of nodes
    - estimated_time_hours: float — Based on 37 km/h average ship speed
    - route_score: float          — A* g_score (lower is better)

CLASS: RouteOptimizer

  __init__(self, graph_state_file):
    - Reads graph_state.json
    - Builds nodes dict: {node_id: {latitude, longitude, overall_risk}}
    - Builds bidirectional adjacency list: {node_id: [{target, distance_km, trade_volume}]}

  haversine_distance(self, lat1, lon1, lat2, lon2) -> float:
    - Computes great-circle distance in km
    - Uses Earth radius = 6371 km

  calculate_heuristic(self, current, goal) -> float:
    - A* heuristic: haversine_distance * (1 + avg_risk)
    - Risk-aware heuristic — penalizes risky areas

  calculate_edge_cost(self, source, target, edge_info, weights) -> float:
    - Multi-objective cost function with 3 components:
      1. distance_cost = distance * weights["distance"]
      2. risk_cost = avg_risk * distance * weights["risk"]
      3. trade_cost = distance * trade_factor * weights["trade"]
         where trade_factor = 1.0 / (1.0 + trade_volume / 1,000,000)
    - total_cost = distance_cost + risk_cost + trade_cost

  find_optimal_route(self, source, destination, weights=None) -> Optional[Route]:
    - A* search algorithm
    - Default weights: risk=1.0, distance=0.3, trade=0.2
    - Tracks g_score, f_score, came_from, distance_to, risk_to
    - Estimates time: total_distance / 37.0 km/h (average vessel speed)
    - Returns Route object or None if no path exists

  find_k_best_routes(self, source, destination, k=3, diversity_penalty=0.3) -> List[Route]:
    - Runs A* with multiple weight configurations:
      Config 1: risk=1.0, distance=0.3, trade=0.2 (risk-heavy)
      Config 2: risk=0.5, distance=0.8, trade=0.2 (distance-heavy)
      Config 3: risk=0.7, distance=0.4, trade=0.5 (balanced with trade)
      Config 4: risk=1.2, distance=0.2, trade=0.1 (very risk-averse)
      Config 5: risk=0.3, distance=1.0, trade=0.3 (shortest path)
    - Diversity check: rejects routes with >70% node overlap with existing routes
    - Returns sorted by route_score (ascending, best first)

  analyze_route(self, route: Route) -> Dict:
    - Produces detailed analysis:
      route_summary: path string, distance, avg_risk, estimated days, score
      node_risks: list of {node, risk} for each node
      high_risk_segments: nodes with overall_risk > 0.7
      bottlenecks: intermediate nodes with ≤2 route alternatives

MAIN BLOCK:
  - Creates sample graph if graph_state.json missing
  - Finds top-3 routes from Hong_Kong to Los_Angeles
  - Prints route details and analysis
  - Exports to optimized_routes.json

================================================================================
7. RISK VECTOR STRUCTURE (6 DIMENSIONS)
================================================================================

Each node carries a 6-dimensional risk vector:

  Index 0: gscpi_risk          — Global Supply Chain Pressure Index (normalized 0-1)
  Index 1: news_risk           — News-derived risk (Claude AI + reporter weight)
  Index 2: political_risk      — Political risk (Claude AI + reporter weight)
  Index 3: trade_risk          — Trade volume risk (inverse — high trade = low risk)
  Index 4: weather_risk        — Weather risk (Claude AI + reporter weight)
  Index 5: reporter_confidence — Mean credibility across all reporter categories

Overall Risk = mean(index 0..4), excluding reporter_confidence.

Risk Level Classification:
  0.0 – 0.3  →  LOW        (normal operations)
  0.3 – 0.6  →  MODERATE   (minor delays possible)
  0.6 – 0.8  →  HIGH       (significant disruptions likely)
  0.8 – 1.0  →  SEVERE     (avoid if possible)


================================================================================
8. KEY ALGORITHMS
================================================================================

8.1  Risk Normalization (Intelligence Processor)
     - GSCPI: Linear normalization with min=0, max=3
     - Trade: Linear normalization (inverted) with min=0, max=10M
     - Text (news/political/weather): Claude AI scoring with reporter credibility weighting
     - Reporter Weight Formula: adjusted = base_risk * credibility + 0.5 * (1 - credibility)

8.2  Graph Attention Network (Graph Risk Engine)
     - 2-layer GAT with multi-head attention
     - Layer 1: 4 heads, 16 hidden dim per head, concatenated → 64 features
     - Layer 2: 1 head, output dim 6, sigmoid activation
     - Edge weights based on inverse distance: 1 / (1 + distance_km / 1000)
     - Bidirectional edges (both directions added)

8.3  A* Pathfinding (Route Optimizer)
     - Heuristic: haversine_distance * (1 + avg_risk) — risk-aware
     - Multi-objective cost: distance_cost + risk_cost + trade_cost
     - K-best routes via multiple weight configurations
     - Diversity filtering: reject routes with >70% node overlap
     - Ship speed assumption: 37 km/h

16. SCRAPER MICROSERVICES LAYER (scrapers/ directory)
================================================================================

The scrapers/ directory contains 6 standalone Flask microservices that replace
the mock data in api_server.py with real, live data from the internet.

Pipeline position:
  [6 Scraper Microservices] → api_server.py → intelligence_processor.py
                            → graph_risk_engine.py → route_optimizer.py

Each scraper runs on its own port and is independently deployable.
They share a common base class (BaseScraper) for SerpAPI, Claude AI,
Flask setup, caching, and logging.


--------------------------------------------------------------------------------
16.1  base_scraper.py  (Shared Infrastructure)
     Purpose: Base class inherited by all 6 scraper agents.
--------------------------------------------------------------------------------

CLASS: BaseScraper

  __init__(self, name: str, port: int, cache_ttl_seconds: int = 1800):
    - Reads SERPAPI_API_KEY and ANTHROPIC_API_KEY from environment
    - Initializes Anthropic client (if key available)
    - Sets up in-memory TTL cache (Dict[str, {"data", "expires_at"}])
    - Creates Flask app with CORS
    - Registers common routes: /health, /api/nodes, /api/cache/clear
    - Sets up colorlog logging

  serpapi_search(self, query: str, search_type: str = "search") -> dict:
    - Wrapper around SerpAPI GoogleSearch
    - search_type "news" adds tbm="nws" param
    - Tracks call count for monitoring
    - Returns parsed results or {} on error

  claude_analyze(self, prompt: str, system: str = None) -> str:
    - Calls Claude claude-sonnet-4-20250514 (max_tokens=500, temperature=0.2)
    - Returns text response or "" on error

  get_cached(self, node_id: str) -> Optional[dict]:
    - Returns cached data if not expired, else None

  set_cached(self, node_id: str, data: dict):
    - Stores data with TTL expiry

  node_to_city(self, node_id: str) -> str:
    - Maps "Hong_Kong" → "Hong Kong", "Los_Angeles" → "Los Angeles"

  utc_timestamp() -> str:  (static)
    - Returns datetime.utcnow().isoformat() + "Z"

  parse_json_from_text(self, text: str) -> dict:
    - Strips markdown code fences, parses JSON
    - Returns {} on parse failure

  run(self):
    - Starts Flask on 0.0.0.0:{port} with threaded=True

CONSTANTS:
  SUPPORTED_NODES = ["Hong_Kong", "Singapore", "Shanghai", "Tokyo", "Los_Angeles"]
  NODE_CITY_MAP = {"Hong_Kong": "Hong Kong", ...}

COMMON ENDPOINTS (all scrapers):
  GET /health          → {"status": "ok", "scraper": "<name>", "port": <port>}
  GET /api/nodes       → {"nodes": ["Hong_Kong", "Singapore", ...]}
  GET /api/cache/clear → {"cleared": true}


--------------------------------------------------------------------------------
16.2  gscpi_agent.py  (Port 8001)
     Purpose: Estimates per-node Global Supply Chain Pressure Index proxy.
--------------------------------------------------------------------------------

CLASS: GSCPIAgent(BaseScraper)

  Cache TTL: 30 minutes
  Fallback value: 1.0 (moderate pressure)

  Endpoint: GET /api/latest/{node_id}
  Response: {"node_id": "...", "value": <float 0.0-3.0>, "timestamp": "..."}

  Algorithm:
    1. SerpAPI search: "{city} port supply chain pressure {month} {year}"
    2. SerpAPI search: "{city} shipping delays freight rates {year}"
    3. Claude synthesizes search results → numeric value in [0.0, 3.0]
    4. Fallback: returns 1.0 if SerpAPI/Claude fail


--------------------------------------------------------------------------------
16.3  news_agent.py  (Port 8002)
     Purpose: Fetches supply-chain-relevant news summaries per port.
--------------------------------------------------------------------------------

CLASS: NewsAgent(BaseScraper)

  Cache TTL: 15 minutes

  Endpoint: GET /api/latest/{node_id}
  Response: {"node_id": "...", "summary": "<max 500 chars>",
             "sources": ["url1", ...], "timestamp": "..."}

  Algorithm:
    1. SerpAPI news search: "{city} port shipping supply chain news"
    2. SerpAPI news search: "{city} logistics disruption strike protest"
    3. Collect top results (title + snippet + link)
    4. Claude summarizes → max 500 chars + source URLs
    5. Fallback: generic "no disruptions" message


--------------------------------------------------------------------------------
16.4  political_agent.py  (Port 8003)
     Purpose: Fetches political risk info relevant to trade/shipping.
--------------------------------------------------------------------------------

CLASS: PoliticalAgent(BaseScraper)

  Cache TTL: 30 minutes

  Endpoint: GET /api/latest/{node_id}
  Response: {"node_id": "...", "report": "<max 500 chars>", "timestamp": "..."}

  Algorithm:
    1. SerpAPI search: "{city} trade sanctions port restrictions political {year}"
    2. SerpAPI search: "{city} protests strikes port workers {month} {year}"
    3. Claude synthesizes → max 500 char report
    4. Fallback: "no active sanctions or restrictions" message


--------------------------------------------------------------------------------
16.5  trade_agent.py  (Port 8004)
     Purpose: Estimates weekly trade volume (USD) per port.
--------------------------------------------------------------------------------

CLASS: TradeAgent(BaseScraper)

  Cache TTL: 60 minutes

  Endpoint: GET /api/latest/{node_id}
  Response: {"node_id": "...", "volume_usd": <float>, "period": "weekly",
             "timestamp": "..."}

  Hardcoded Baseline Volumes (USD, weekly):
    Hong_Kong: 6,500,000   Singapore: 7,200,000   Shanghai: 9,100,000
    Tokyo: 5,800,000       Los_Angeles: 6,000,000

  Algorithm (3-tier fallback):
    1. UN Comtrade public API (with exponential backoff retries)
    2. SerpAPI + Claude estimation (adjusts from baseline)
    3. Hardcoded baseline as ultimate fallback


--------------------------------------------------------------------------------
16.6  weather_agent.py  (Port 8005)
     Purpose: Current weather and maritime alerts per port.
--------------------------------------------------------------------------------

CLASS: WeatherAgent(BaseScraper)

  Cache TTL: 10 minutes

  Endpoint: GET /api/latest/{node_id}
  Response: {"node_id": "...", "conditions": "<max 300 chars>",
             "alerts": [{type, severity, description, expires}],
             "timestamp": "..."}

  Port Coordinates:
    Hong_Kong: 22.3193, 114.1694   Singapore: 1.3521, 103.8198
    Shanghai: 31.2304, 121.4737    Tokyo: 35.6762, 139.6503
    Los_Angeles: 33.7288, -118.2620

  Algorithm:
    1. Open-Meteo API (free, no key) — temperature, wind, weather code, precip
    2. Optional: OpenWeatherMap (if OPENWEATHERMAP_API_KEY set)
    3. SerpAPI for weather alerts (typhoon, storm, fog, tsunami keywords)
    4. WMO weather-code → human-readable mapping
    5. Detects alert keywords in search results → structured alerts array

  WMO Codes: Full mapping from code 0 (Clear sky) through 99 (Heavy hail)


--------------------------------------------------------------------------------
16.7  reporter_agent.py  (Port 8006)
     Purpose: Scores reliability of data sources per node.
--------------------------------------------------------------------------------

CLASS: ReporterAgent(BaseScraper)

  Cache TTL: 60 minutes

  Endpoint: GET /api/credibility/{node_id}
  Response: {"node_id": "...", "scores": {"news": 0.82, "political": 0.75,
             "weather": 0.91}, "updated_at": "..."}

  Base Credibility Priors:
    Hong_Kong:   news=0.80, political=0.70, weather=0.90
    Singapore:   news=0.85, political=0.80, weather=0.92
    Shanghai:    news=0.70, political=0.60, weather=0.88
    Tokyo:       news=0.88, political=0.85, weather=0.95
    Los_Angeles: news=0.90, political=0.87, weather=0.95

  Dynamic Adjustment:
    - Probes news (8002), political (8003), weather (8005) scrapers
    - If multiple sources found → boost credibility by +0.10
    - If thin data or failure → lower credibility by -0.15
    - Scores clamped to [0.0, 1.0]

16.8  run_all.py  (Launcher)
     Purpose: Starts all 6 scrapers as subprocesses.
--------------------------------------------------------------------------------

  Usage: python scrapers/run_all.py
  Ctrl+C gracefully terminates all subprocesses.
  Also runnable individually: python scrapers/<agent>.py

  Start order: GSCPI(8001), News(8002), Political(8003),
               Trade(8004), Weather(8005), Reporter(8006)
  0.5s stagger between launches to avoid port conflicts.
  Monitors for unexpected subprocess exits.

================================================================================

    gscpi_data["value"]          → ScraperData.gscpi

    reporter_data["scores"]      → ScraperData.reporter_credibility
